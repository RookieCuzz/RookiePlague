# 动物配置表示例

## 表格结构说明

配置表采用 Excel 格式（`animal.xlsx`），包含以下字段：

| 字段名 | 类型 | 分组 | 说明 |
|--------|------|------|------|
| type | string | c,s | 动物类型（对应原版动物类型，如COW、PIG、CHICKEN等） |
| desc | string | c,s | 动物描述（中文名称） |
| speciesFactor | float | c,s | 物种因子（影响染疫概率计算的系数） |
| chunkLimit | int | c,s | 区块上限（同类动物在单个区块内的数量上限） |
| corpseDropRate | int | c,s | 死亡生成尸体概率（百分比，如80表示80%） |
| corpseMobid | string | c,s | 尸体的CE模型id（对应CustomEntity插件的模型ID） |
| maxBreedTimes | int | c,s | 最大繁殖次数（每只动物的繁殖次数上限） |
| plagueDeathTime | int | c,s | 瘟疫致死时间（未治疗情况下从感染到死亡的时间，单位：秒） |

## 配置示例

### 🐑 羊 (SHEEP)

| 字段 | 值 |
|------|-----|
| **type** | SHEEP |
| **desc** | 羊 |
| **speciesFactor** | 0.9 |
| **chunkLimit** | 18 |
| **corpseDropRate** | 70 |
| **corpseMobid** | animal_corpse_large |
| **maxBreedTimes** | 5 |
| **plagueDeathTime** | 320 |

**配置说明：**
- 物种因子为0.9，染疫概率较低
- 每个区块最多18只羊
- 死亡后有70%概率生成尸体
- 使用大型动物尸体模型
- 最多可繁殖5次
- 感染瘟疫后320秒（约5.3分钟）死亡

## 配置规则

### 物种因子 (speciesFactor)
- **范围**: 0.1 - 2.0
- **说明**: 数值越大，该物种染疫概率越高
- **建议**: 
  - 大型动物（牛、羊）: 0.8 - 1.0
  - 中型动物（猪）: 1.0 - 1.2
  - 小型动物（鸡）: 1.2 - 1.5

### 区块上限 (chunkLimit)
- **说明**: 限制单个区块内同类动物数量，防止过度繁殖
- **建议**:
  - 大型动物: 10-15只
  - 中型动物: 15-20只
  - 小型动物: 18-25只

### 尸体掉落概率 (corpseDropRate)
- **范围**: 0 - 100
- **说明**: 动物死亡时生成尸体实体的概率（百分比）
- **建议**: 50-80之间，可根据游戏平衡性调整

### 最大繁殖次数 (maxBreedTimes)
- **说明**: 限制每只动物的总繁殖次数
- **建议**:
  - 大型动物: 3-5次
  - 中型动物: 4-6次
  - 小型动物: 6-10次

### 瘟疫致死时间 (plagueDeathTime)
- **单位**: 秒
- **建议**:
  - 快速致死: 180-240秒（3-4分钟）
  - 中等速度: 300-360秒（5-6分钟）
  - 缓慢致死: 420-600秒（7-10分钟）

## 使用说明

1. 在 `animal.xlsx` 中按照表格结构添加配置
2. 第1行为字段标识（##var）
3. 第2行为字段类型（##type）
4. 第3行为分组信息（##group）
5. 第4行为中文注释（##）
6. 从第5行开始为实际配置数据
7. 使用 Luban 工具生成最终配置文件
